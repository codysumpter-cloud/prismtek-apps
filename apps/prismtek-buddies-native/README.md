# Prismtek Buddies (native, cozy room v0)

A small native SwiftUI app for macOS and iOS: a cozy productivity room with Bitbud, a
focus timer, tasks, a memo pad, ambience toggles, a mini mode, and an XP/progression stub.

Original SwiftUI art only. The only bitmap asset shipped is Bitbud, sliced from Cody's own
`bitbud` pet atlas (his asset). No third-party packs, no PixelLab, no network image gen.

## Layout

```
apps/prismtek-buddies-native/
  project.yml                          # XcodeGen project (two targets, shared sources)
  scripts/extract_bitbud_frames.py     # reproducible Pillow slicer
  Shared/                              # SwiftUI views, models, renderer, Resources
    Resources/BitbudFrames/*.png       # 57 extracted frames (committed)
    Resources/bitbud-pet.json          # provenance copy of the Bitbud manifest
  MacApp/                              # macOS @main + Info.plist
  iOSApp/                              # iOS @main + Info.plist
```

## Toolchain note (important)

Xcode 27 is NOT the default toolchain on this machine. Every `xcodebuild` call must be
prefixed with `DEVELOPER_DIR`:

```
DEVELOPER_DIR=/Applications/Xcode-beta.app/Contents/Developer xcodebuild ...
```

SDKs: `macosx27.0`, `iphonesimulator27.0`.

## Generate the project

The `.xcodeproj` is committed (repo convention — matches `apps/bemore-macos-native`), but
it is generated by XcodeGen. Regenerate after editing `project.yml`:

```
cd apps/prismtek-buddies-native
xcodegen generate
```

## Build

macOS:

```
DEVELOPER_DIR=/Applications/Xcode-beta.app/Contents/Developer xcodebuild \
  -project PrismtekBuddies.xcodeproj -scheme PrismtekBuddiesMac \
  -sdk macosx27.0 -configuration Debug CODE_SIGNING_ALLOWED=NO build
```

iOS (simulator):

```
DEVELOPER_DIR=/Applications/Xcode-beta.app/Contents/Developer xcodebuild \
  -project PrismtekBuddies.xcodeproj -scheme PrismtekBuddiesiOS \
  -sdk iphonesimulator27.0 -destination 'generic/platform=iOS Simulator' \
  CODE_SIGNING_ALLOWED=NO build
```

## Open in Xcode

```
open PrismtekBuddies.xcodeproj
```

Pick `PrismtekBuddiesMac` (My Mac) or `PrismtekBuddiesiOS` (a simulator) and run. No
DEVELOPMENT_TEAM is set; for device runs you must add your own signing.

## Where Bitbud loads from

`BitbudRenderer` loads `Shared/Resources/BitbudFrames/<state>_<n>.png` (bundled as app
resources) and cycles them on a `TimelineView` timer. It never decodes WebP at runtime.

Frames are produced by `scripts/extract_bitbud_frames.py`, which reads the READ-ONLY
source atlas at `/Users/prismtek/.codex/pets/bitbud/spritesheet.webp` (1536x1872, 8x9 grid,
192x208 cells) and writes 57 PNGs. The source webp is never mutated or committed.

To re-slice:

```
python3 scripts/extract_bitbud_frames.py
```

## Features (v0)

- Cozy room scene (original SwiftUI shapes/gradients): wall, floor, desk, shelf, window,
  rug, pet zone.
- Bitbud renderer with state mapping: idle / waving / waiting / running / review / failed /
  jumping.
- Focus timer: Pomodoro (25/5) + count-up, start/pause/reset. Completing a focus block
  grants XP and triggers a jump.
- Tasks: add / mark done / delete, persisted via `@AppStorage` JSON.
- Memo: free-text pad, persisted via `@AppStorage`.
- Ambience toggles (rain/keyboard/fireplace/café): UI state only — **audio is a silent
  placeholder in v0** (no bundled audio to keep licensing clean).
- Mini mode: macOS opens a small floating always-on-top `NSWindow`; iOS uses the compact
  tabbed layout.
- Progression: focus XP + level + gift-unlock placeholder banner.

## Known limitations

- Ambience audio is a placeholder (no sound files shipped).
- Gift unlocks show a banner only — no actual reward content yet.
- iOS mini mode is the compact tab layout, not a floating window (no equivalent on iOS).
